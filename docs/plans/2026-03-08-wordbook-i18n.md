# Implementation Plan: Fix Wordbook Performance & Add i18n

**Date:** 2026-03-08
**Status:** Draft

## Problem Statement

1. **Wordbook Hang**: VocabularyView hangs/freezes when navigating to the tab
2. **Missing i18n**: All UI strings are hardcoded in Chinese, need internationalization

## Root Cause Analysis

### Wordbook Performance Issue

Potential causes:
- `filteredVocabulary` computed property sorts entire vocabulary array on every access
- List rendering all items synchronously without lazy loading
- No pagination or virtualization for large vocabulary lists
- Search filtering happens on main thread

### i18n Gap

- All strings hardcoded in Chinese throughout the codebase
- No localization infrastructure in place
- Need to support multiple languages (English, Chinese at minimum)

## Proposed Solution

### Part 1: Fix Wordbook Performance

**Approach:** Optimize the filtering and sorting logic

1. Cache sorted vocabulary to avoid re-sorting on every render
2. Use `@State` to store filtered results instead of computed property
3. Debounce search input to reduce filtering frequency
4. Add lazy loading if vocabulary count is very large

**Files to modify:**
- `CelestialPDFs/Views/VocabularyView.swift`

### Part 2: Add i18n Support

**Approach:** Use Xcode String Catalog (.xcstrings) - modern, recommended approach

1. Create String Catalog for localized strings
2. Extract all hardcoded strings to LocalizedStringKey
3. Add English and Chinese localizations
4. Update all View files with localized strings

**Files to modify:**
- Create: `CelestialPDFs/Localizable.xcstrings`
- Update: All View files (VocabularyView, BookshelfView, PDFReaderView, etc.)
- Update: AIService.swift (error messages)
- Update: BookStore.swift (user-facing messages)

## Implementation Steps

### Step 1: Fix VocabularyView Performance

**Changes:**
- Replace computed `filteredVocabulary` with `@State` cached array
- Add `updateFilteredVocabulary()` method called on appear and search change
- Use `Task` to perform filtering off main thread for large lists

**Expected outcome:** Instant navigation to vocabulary tab, smooth search

### Step 2: Create String Catalog

**Changes:**
- Add `Localizable.xcstrings` to project
- Define base strings in Chinese (existing language)
- Add English translations

**Expected outcome:** Localization infrastructure ready

### Step 3: Localize VocabularyView

**Changes:**
- Replace hardcoded strings with `LocalizedStringKey`
- Example: `"单词本"` → `"vocabulary.title"`

**Expected outcome:** VocabularyView supports multiple languages

### Step 4: Localize Remaining Views

**Changes:**
- BookshelfView, PDFReaderView, AIChatView, etc.
- All user-facing strings converted to localized keys

**Expected outcome:** Full app localization

### Step 5: Localize Services

**Changes:**
- AIService error messages
- BookStore status messages

**Expected outcome:** All user-facing text localized

4. **Fallback:** Verify Chinese strings show when English not available
